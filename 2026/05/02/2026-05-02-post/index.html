1.Hello, World!

·JavaScript 在哪儿写、怎么运行？

JS需要个地方运行——用浏览器最方便。

怎么把 JS 塞进网页里——靠一个叫 <script> 的标签。

对于服务器端环境（如 Node.js），你只需要使用诸如 “node my.js” 的命令行来执行它。

·关于 <script> 标签的那些“老古董”特性

<script> 标签有一些现在很少用到的特性（attribute），但是我们可以在老代码中找到它们：

type特性：<script type=…>:

在老的 HTML4 标准中，要求 script 标签有 type 特性。通常是 type=”text/javascript”。这样的特性声明现在已经不再需要。而且，现代 HTML 标准已经完全改变了此特性的含义。现在，它可以用于 JavaScript 模块。

language 特性：<script language=…>:

这个特性是为了显示脚本使用的语言。这个特性现在已经没有任何意义，因为语言默认就是 JavaScript。不再需要使用它了。

脚本前后的注释:

在非常古老的书籍和指南中，你可能会在 <script> 标签里面找到注释，就像这样：


    <script type="text/javascript"><!--

        ...

    //--></script>

现代 JavaScript 中已经不这样使用了。这些注释是用于不支持 <script> 标签的古老的浏览器隐藏 JavaScript 代码的。由于最近 15 年内发布的浏览器都没有这样的问题，因此这种注释能帮你辨认出一些老掉牙的代码。

·把 JS 代码放到单独文件里

当 JS 代码变长，不可能全写在 HTML 里，不然乱死了。

所以可以把代码放在一个 .js 文件里，再用 <script> 的 src 属性引入

<script src="/path/to/script.js"></script>

这里，/path/to/script.js 是脚本文件从网站根目录开始的绝对路径。当然也可以提供当前页面的相对路径。例如，src=”script.js”，就像 src=”./script.js”，表示当前文件夹中的 “script.js” 文件。

我们也可以提供一个完整的 URL 地址，例如：

<script src="https://example.com/script.js"></script>
如果用了 src，标签里面不能再写任何代码，写了也会被忽略。

正确写法：分成两个标签


    <script src="my-script.js"></script>
    <script>
    alert(1);
    </script>

·好处是什么？

  • 代码复用——多个页面可以共用同一个 JS 文件。
  • 加载更快——浏览器会把 JS 文件缓存下来，第二次访问不用重新下载。

2.代码结构

·什么是“语句”？

语句是执行行为（action）的语法结构和命令。

就是一句一句的命令，告诉浏览器要干什么。

比如 alert(‘Hello’) 就是一条语句，意思是“弹出一个写着 Hello 的框”。

你可以写很多条语句，它们会从上到下依次执行。


    alert('Hello');
    alert('World');

·分号 ; 到底要不要写？

结论：新手一定要写分号！

虽然 JS 有个“自动插入分号”的机制，换行有时可以代替分号，但不总是靠谱。不写分号可能在某些情况下产生难以发现的 bug。

自动分号插入:
当存在换行符（line break）时，在大多数情况下可以省略分号。
下面的代码也是可以运行的：


  alert('Hello')
  alert('World')

在这，JavaScript 将换行符理解成“隐式”的分号。

举个例子（你暂时不用懂细节，只记住现象就行）：

没写分号的坑：


    alert("Hello")  // 这里没分号

    [1, 2].forEach(alert);

这段代码不会弹出 1 和 2，只会弹 Hello，并且控制台会报错。

因为 JS 把两行当成一行了，变成了 alert(“Hello”)[1,2].forEach(alert)，这完全是错的理解。

正确写法（加了分号）：


    alert("Hello");  

    [1, 2].forEach(alert);

·注释

注释是写给你自己（或别人）看的说明，浏览器会完全忽略它们。

单行注释：//

多行注释：/* … */

实用技巧：用注释临时禁用一段代码

快捷键：在大多数代码编辑器里，选中代码按 Ctrl + /（Mac 是 Cmd + /）可以快速把代码变成注释，再按一次取消注释。多行注释快捷键通常是 Ctrl + Shift + /。
注释不能嵌套：别在一个 /…/ 里面再写一个 /…/，会报错。

3.现代模式，”use strict”

·为什么需要 “use strict”？

长久以来，JavaScript 不断向前发展且并未带来任何兼容性问题。新的特性被加入，旧的功能也没有改变。

这么做有利于兼容旧代码，但缺点是 JavaScript 创造者的任何错误或不完善的决定也将永远被保留在 JavaScript 语言中。

这种情况一直持续到 2009 年 ECMAScript 5 (ES5) 的出现。ES5 规范增加了新的语言特性并且修改了一些已经存在的特性。为了保证旧的功能能够使用，大部分的修改是默认不生效的。你需要一个特殊的指令 —— “use strict” 来明确地激活这些特性。

·怎么开启严格模式？

放在脚本最顶部（推荐新手）


    "use strict";

    // 从这里开始，下面的代码都在严格模式下运行
    alert("Hello");

放在函数内部（只对该函数有效）


    function myFunction() {
    "use strict";
    // 只有这个函数内部是严格模式
    }
1."use strict" 必须在脚本的最顶部:如果前面有别的代码，它就不生效了（注释除外）。 2.一旦开启严格模式，就关不掉了。

·浏览器控制台

你在网页里按 F12 打开的控制台（Console）里直接敲代码，默认是不开启严格模式的。

有时，当 use strict 会对代码产生一些影响时，你会得到错误的结果。

如果你想在控制台里测试严格模式下的代码，可以这样：

方法一（多行输入）

先输入 ‘use strict’; 然后按 Shift + Enter 换行，再写后面的代码，最后按 Enter 执行。

如果不行，例如你使用的是旧版本的浏览器，那么有一种很丑但可靠的启用 use strict 的方法。

方法二（包装一下）

把代码包在一个自执行函数里：


    (function() {
      'use strict';
      // 你的代码
    })();

·我们应该使用 “use strict” 吗？

新手答案：写！每次都在文件开头写。

等你以后学了 class 和 module（高级内容），它们会自动开启严格模式，那时候你就不用手动写了。

4.变量

·变量是什么？

大多数情况下，JavaScript 应用需要处理信息。这有两个例子：

一个网上商店 —— 这里的信息可能包含正在售卖的商品和购物车。

一个聊天应用 —— 这里的信息可能包括用户和消息等等。

变量就是用来储存这些信息的。

变量是数据的“命名存储”。我们可以使用变量来保存商品、访客和其他信息。

·怎么创建一个变量？

用关键字 let。


    let message;           // 声明一个叫 message 的盒子（里面暂时没东西）
    message = 'Hello!';    // 把 'Hello!' 放进去

    alert(message);        // 弹出盒子里面的东西 -> Hello!

也可以一步到位：


    let message = 'Hello!';   // 声明 + 放东西一起做

可以在一行中声明多个变量：


  let user = 'John', age = 25, message = 'Hello';

看上去代码长度更短，但并不推荐这样。为了更好的可读性，请一行只声明一个变量。多行变量声明有点长，但更容易阅读
一些程序员采用下面的形式书写多个变量：


let user = 'John',
  age = 25,
  message = 'Hello';

甚至使用“逗号在前”的形式：


let user = 'John'
  , age = 25
  , message = 'Hello';

技术上讲，这些变体都有一样的效果。所以，这是个个人品味和审美方面的问题。

·盒子里的东西可以换


    let message = 'Hello!';
    message = 'World!';    // 换东西了，原来的 'Hello!' 被扔掉

    alert(message);        // 现在弹出 World!

·一个变量只能声明一次

错误写法（会报错）：


    let message = 'Hello';
    let message = 'World';   // SyntaxError: 变量已经声明过了

正确写法：


    let message = 'Hello';
    message = 'World';       // 不用再写 let，直接赋值就行

有趣的是，也存在禁止更改变量值的 函数式 编程语言。比如 Scala 或 Erlang。
在这种类型的语言中，一旦值保存在盒子中，就永远存在。如果你试图保存其他值，它会强制创建一个新盒子（声明一个新变量），无法重用之前的变量。
虽然第一次看上去有点奇怪，但是这些语言有很大的发展潜力。不仅如此，在某些领域，比如并行计算，这个限制有一定的好处。研究这样的一门语言（即使不打算很快就用上它）有助于开阔视野。

·命名规则

  • 只能用 字母、数字、$、_。
  • 不能以数字开头。
  • 区分大小写（apple 和 Apple 是两个不同盒子）。
  • 不能用 JS 的保留字（比如 let、class、return。参照保留字列表）。
  • 技术上允许中文，但强烈不推荐，为了其他国家的人的阅读

命名习惯：驼峰式命名法（camelCase）
第一个单词小写，后面每个单词首字母大写。
例：myVeryLongName、userAge、shoppingCartTotal。

未采用 use strict 下的赋值
一般，我们需要在使用一个变量前定义它。但是在早期，我们可以不使用 let 进行变量声明，而可以简单地通过赋值来创建一个变量。现在如果我们不在脚本中使用 use strict 声明启用严格模式，这仍然可以正常工作，这是为了保持对旧脚本的兼容。


   // 注意：这个例子中没有 "use strict"

   num = 5; // 如果变量 "num" 不存在，就会被创建

   alert(num); // 5

上面这是个糟糕的做法，严格模式下会报错。


   "use strict";

   num = 5; // 错误：num 未定义

·常量

有些数据你不想让它被改动，比如生日、数学常数，就用 const 声明。


    const myBirthday = '18.04.1982';
    myBirthday = '01.01.2001';   // 报错！常量不能重新赋值
什么时候用 const？ 
当程序员能确定这个变量永远不会改变的时候，就可以使用 const 来确保这种行为，并且清楚地向别人传递这一事实。

·大写形式的常数

一个普遍的做法是将常量用作别名，以便记住那些在执行之前就已知的难以记住的值。

使用大写字母和下划线来命名这些常量。


    const COLOR_RED = "#F00";
    const COLOR_GREEN = "#0F0";
    const COLOR_BLUE = "#00F";
    const COLOR_ORANGE = "#FF7F00";

    // ……当我们需要选择一个颜色
    let color = COLOR_ORANGE;
    alert(color); // #FF7F00

但如果值是在程序运行中计算出来、但之后不会改的（比如页面加载时间），仍然用 const，但命名用小写驼峰。


    const pageLoadTime = /* 网页加载所需的时间 */;

pageLoadTime 的值在页面加载之前是未知的，所以采用常规命名。但是它仍然是个常量，因为赋值之后不会改变。

换句话说，大写命名的常量仅用作“硬编码（hard-coded）”值的别名。

·正确命名变量的一些可以遵循的规则

  • 使用易读的命名，比如 userName 或者 shoppingCart。
  • 离诸如 a、b、c 这种缩写和短名称远一点，除非你真的知道你在干什么。
  • 变量名在能够准确描述变量的同时要足够简洁。不好的例子就是 data 和 value，这样的名称等于什么都没说。如果能够非常明显地从上下文知道数据和值所表达的含义，这样使用它们也是可以的。
  • 脑海中的术语要和团队保持一致。如果网站的访客称为“用户”，则我们采用相关的变量命名，比如 currentUser 或者 newUser，而不要使用 currentVisitor 或者一个 newManInTown。
宁可多声明几个变量，也不要一个变量反复装不同类型的东西（重用现有的变量），那样会把人搞晕。

5.数据类型

JavaScript 中的值都具有特定的类型。例如，字符串或数字。

在 JavaScript 中有 8 种基本的数据类型（译注：7 种原始类型和 1 种引用类型）。

我们可以将任何类型的值存入变量。例如，一个变量可以在前一刻是个字符串，下一刻就存储一个数字：


    // 没有错误
    let message = "hello";
    message = 123456;

允许这种操作的编程语言，例如 JavaScript，被称为“动态类型”（dynamically typed）的编程语言，意思是虽然编程语言中有不同的数据类型，但是你定义的变量并不会在定义后，被限制为某一数据类型。
