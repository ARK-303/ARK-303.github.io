1. JavaScript 简介

·JavaScript 是什么？

最初目的：让网页“活”起来。比如，你点一个按钮，网页会弹出提示、切换图片、加载新内容……这些“动”的效果，很多就是 JavaScript 实现的。

怎么运行：JavaScript 代码（也叫“脚本”）可以直接写在网页文件里，浏览器打开网页时，它就会自动执行。不需要特殊的准备或编译即可运行。

·JavaScript 在哪里运行？

主要在浏览器里。

也能在服务器上运行（比如用 Node.js），甚至能在任何安装了 JavaScript 引擎的设备上运行。

引擎：负责读懂 JavaScript 代码，然后转换成电脑能执行的指令(机器语言)，让代码跑起来。
常见的引擎有 Chrome Edge用的 V8、Firefox 用的 SpiderMonkey​ 等。

·JavaScript 能做什么？

简单说，能让网页和你互动，能改变网页内容，能跟服务器通信。比如：

  • 改网页：动态添加、删除、修改网页上的文字、图片、样式。(修改HTML)
  • 响应用户：你点鼠标、按键盘、移动鼠标，网页能做出反应（比如弹出菜单、验证表单）。
  • 和服务器通信：不用刷新整个网页，就能从服务器加载新数据（比如刷微博时，不断看到新内容）(AJAX和COMET技术)。
  • 记住东西：在你的浏览器里存一点数据（比如记住你的登录状态、购物车里的商品）。(本地存储客户端数据，即cookie)
  • 操作文件：在用户允许的情况下，可以处理用户上传的文件（比如上传头像）。

不提供对内存或 CPU 的底层访问,它的能力很大程度上取决于它运行的环境。

·JavaScript 不能做什么？

  • 不能访问你的隐私数据：比如通讯录、短信、位置等，除非你明确授权。
  • 不能随便动你的电脑上的文件：不能偷偷读、写、删你硬盘里的东西。只有在你自己操作时（比如拖拽文件到浏览器、点击上传按钮），它才能处理你指定的那个文件。
  • 不能用你的摄像头/麦克风偷窥偷听：必须经过你明确的同意和授权。
  • 不能随便访问其他网站的页面：比如你打开了 A 网站，它不能去偷看你在另一个标签页里打开的 B 网站（比如你的邮箱）的内容。这是“同源策略”，是为了保护你的隐私和安全。
注意：上面这些限制主要针对浏览器环境。如果 JavaScript 运行在服务器上（比如 Node.js），这些限制很多就不存在了。

·JavaScript 为什么这么流行、这么特别？

主要有三点：

  • 和 HTML/CSS 是天生的搭档：网页的结构（HTML）、样式（CSS）、行为（JavaScript）三者里，只有 JavaScript 能完美地操控前两者，让网页真正“动”起来。
  • 简单的事，简单做：很多常见的网页交互功能，用 JavaScript 实现起来比较直接。
  • 所有主流浏览器都支持：只要是个现代浏览器，都默认支持 JavaScript。你不用让用户额外安装东西。

此外，JavaScript 还可用于创建服务器和移动端应用程序等。

·JavaScript “上层”语言

不同的人想要不同的功能。JavaScript 的语法也不能满足所有人的需求。

这是正常的，因为每个人的项目和需求都不一样。

因此，最近出现了许多新语言，这些语言在浏览器中执行之前，都会被 编译（转化）成 JavaScript。

现代化的工具使得编译速度非常快且透明，实际上允许开发者使用另一种语言编写代码并会将其“自动转换”为 JavaScript。

此类语言的示例有：

  • CoffeeScript 是 JavaScript 的一种语法糖。它引入了更加简短的语法，使我们可以编写更清晰简洁的代码。通常，Ruby 开发者喜欢它。
  • TypeScript 专注于添加“严格的数据类型”以简化开发，以更好地支持复杂系统的开发。由微软开发。
  • Flow 也添加了数据类型，但是以一种不同的方式。由 Facebook 开发。
  • Dart 是一门独立的语言。它拥有自己的引擎，该引擎可以在非浏览器环境中运行（例如手机应用），它也可以被编译成 JavaScript。由 Google 开发。
  • Brython 是一个 Python 到 JavaScript 的转译器，让我们可以在不使用 JavaScript 的情况下，以纯 Python 编写应用程序。
  • Kotlin 是一个现代、简洁且安全的编程语言，编写出的应用程序可以在浏览器和 Node 环境中运行。

2. 手册与规范

这段原文，其实是在给你介绍三件“法宝”。这三件法宝不是让你一开始就啃的，而是告诉你“当你有需要时，该去哪里找最权威的答案”。

这三件法宝是什么？

想象一下，你要组装一个复杂的乐高：

终极设计图（规范）：乐高公司出的官方、精确到每一个颗粒的图纸。绝对权威，但全是冰冷的图纸语言，新手看了头大。

步骤详解视频（MDN手册）：热心高手录制的视频，会告诉你“这一步怎么拼，为什么要这么拼，还有没有别的拼法”。这是你最常用、最该依赖的学习工具。

零件兼容清单（兼容性表）：告诉你，某个新出的、很酷的零件，在你的旧款乐高套装里能不能用。避免你辛辛苦苦设计好，却发现根本拼不上。

·ECMAScript 规范

它是什么：JavaScript语言的官方标准说明书。由ECMA国际组织制定，规定了JavaScript这门语言到底应该是什么样子、怎么工作。所有浏览器和JavaScript引擎（如V8）最终都以它为准绳来实现功能。

为什么你需要知道它：作为新手，你几乎完全不需要直接看它。它的语言极其严谨、抽象，像法律条文。

对你现在有什么用：你只需要记住有这个权威存在就行了。

每年都会发布一个新版本的规范。最新的规范草案请见 https://tc39.es/ecma262/。
想了解最新最前沿的功能，包括“即将纳入规范的”（所谓的 “stage 3”），请看这里的提案 https://github.com/tc39/proposals。

·MDN JavaScript 手册

它是什么：这是你未来最亲密、最可靠的伙伴！你可以把它看作全球Web开发者共同维护的、关于前端技术（HTML, CSS, JavaScript）的“维基百科”或“新华字典”，但质量极高，由Mozilla等机构维护。

为什么你需要它：教程教给你基础方法和思路，但不可能记住所有细节。当你在写代码时，突然想不起来array.map()具体怎么用、parseInt()第二个参数是什么，或者想了解Promise的所有方法时，MDN就是你的即时查询手册。

你怎么用它：
直接访问：https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference

最佳用法：善用搜索：在Google或Bing里搜“MDN”加关键词。比如，下次你不懂parseInt，不要去搜“parseInt怎么用”，而是直接搜 “MDN parseInt”​ 。这样通常第一个结果就是最权威、最清晰的MDN文档，里面会有语法、参数解释、示例代码和注意事项。这是专业开发者的标准操作。

·浏览器兼容性表

它是什么：一张张表格，告诉你某个JavaScript新功能（新“零件”）在不同的浏览器（Chrome, Firefox, Safari, Edge）及其不同版本中是否被支持。

为什么你需要它：JavaScript在不断更新，每年都有新语法、新API（应用程序编程接口）加入。但用户的浏览器可能很旧（比如IE浏览器，或很久没更新的手机浏览器）。你不能在代码里使用一个只有最新Chrome才支持的功能，然后指望所有用户的旧手机Safari都能运行。否则网站会出错。

你怎么用它：

Can I use：https://caniuse.com 它主要查Web API（如加密、文件操作、CSS新属性等）的兼容性。比如你想用“现代加密函数”，就去这里查。

(每个功能的支持表，例如，查看哪个引擎支持现代加密（cryptography）函数)

Compat Table：https://kangax.github.io/compat-table 它主要查JavaScript语言本身的新语法（如ES6, ES7等新特性）的兼容性。

(一份列有语言功能以及引擎是否支持这些功能的表格。)

对新手来说：刚开始学基础语法时，你可能用不上它，因为你在学的东西已经被所有现代浏览器支持很多年了。但当你开始做真正的项目，尤其想尝试一些炫酷的新功能时，一定要先来这些网站查一下，看看你的目标用户群用的浏览器是否支持。这是一个好习惯。

3.代码编辑器

选代码编辑器，就像选你写字用的笔和本子，目的是让你写代码更顺手、更高效。
主要分两种：IDE（集成开发环境）和轻量编辑器。很多人喜欢这两种各选一个。(全能工作站 vs 轻便笔记本)

·IDE（集成开发环境）

它管整个项目：是个完整的“开发环境”。

功能强大：IDE 加载项目（通常包含很多文件），并且允许在不同文件之间导航（navigation）。IDE 还提供基于整个项目（不仅仅是打开的文件）的自动补全功能，集成版本控制（如 git）、集成测试环境等一些其他“项目层面”的东西。

如果你还没考虑好选哪一款 IDE，可以考虑下面两个：
Visual Studio Code（跨平台，免费）。
WebStorm（跨平台，非商用免费）。
对于 Windows 系统来说，也有个叫 “Visual Studio” 的 IDE，请不要跟 “Visual Studio Code” 混淆。“Visual Studio” 是一个收费的、强大的 Windows 专用编辑器，它十分适合于 .NET 开发。用它进行 JavaScript 开发也不错。
“Visual Studio” 有个免费的版本 Visual Studio Community。

·轻量编辑器

它主要管单个文件：快速打开、编辑、保存单个文件。

特点：启动极快，界面简洁，通过安装插件也能获得类似IDE的很多功能（比如代码提示）。

界限模糊：现在很多轻量编辑器装上插件后，功能不输IDE。

Sublime Text（跨平台，共享软件）。
Notepad++（Windows，免费）。
Vim 和 Emacs 也很棒，如果你知道怎么使用它们的话。

4.开发者控制台

核心问题：代码出错了怎么办？

人写代码一定会出错，这100%正常，就像学写字一定会写错别字一样。关键不是不犯错，而是知道怎么找到错误、看懂错误、改掉错误。

解决办法：浏览器为我们（开发者）准备了一个“后台管理工具”，叫开发者工具。我们可以用它来“看后台报错”。

通常，开发者倾向于使用 Chrome 或 Firefox 进行开发，因为它们有最好的开发者工具。一些其它的浏览器也提供开发者工具，有时还具有一些特殊的功能，通常它们都是在“追赶” Chrome 或 Firefox。所以大多数人都有“最喜欢”的浏览器，当遇到某个浏览器独有的问题的时候，人们就会切换到其它的浏览器。

·Google Chrome

打开网页 bug.html。

按下 F12 键，如果你使用 Mac，试试 Cmd+Opt+J。

默认情况下，开发者工具会被在 Console 标签页中打开。

在这我们能看到红色的错误提示信息。这个场景中，脚本里有一个未知的 “lalala” 命令。

在右边，有个可点击的链接 bug.html:12。这个链接会链接到错误发生的行号。

在错误信息的下方，有个 > 标志。它代表“命令行”，在“命令行”中，我们可以输入 JavaScript 命令，按下 Enter 来执行。

·Firefox、Edge 和其它浏览器

大多数其它的浏览器都是通过 F12 来打开开发者工具。它们的外观和感觉都非常相似。

·Safari

Safari（Mac 系统中的浏览器，Windows 和 Linux 系统不支持）有一点点不同。我们需要先开启“开发菜单”。

打开“偏好设置”，选择“高级”选项。选中最下方的那个选择框：

Show Develop menu in menu bar（在菜单栏显示开发菜单）。

现在，我们通过 Cmd+Opt+C 就能打开或关闭控制台了。另外注意，有一个名字为“开发”的顶部菜单出现了。它有很多命令和选项。
